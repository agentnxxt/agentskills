Installation

Every skill is distributed as a .skill file (a ZIP archive containing SKILL.md and reference materials). Install in 3 steps:

1. Download

Grab the .skill file from the marketplace or clone the repo:

git clone https://github.com/agentnxxt/agentskills.git

2. Add to Claude

In Claude Desktop or claude.ai, use the /skill command or drag the .skill file into the conversation. Claude will read the SKILL.md and activate the persona.

3. Trigger

Each skill has natural language triggers. Just ask Claude what the skill does — it will activate automatically. See individual skill docs below for specific trigger phrases.

Architecture

Every skill follows the same structure:

autonomyx-{skill-name}/
  SKILL.md                    # Core skill definition + persona
  references/
    autonomyx-standard.md     # Shared boilerplate (feedback, versioning, CTA)
    {additional-refs}.md      # Skill-specific reference data

Skill Anatomy

  • Frontmatter — YAML metadata: name, version, author, description, trigger phrases
  • Persona — Role definition ("You are a senior brand designer at...")
  • Workflow — Multi-step process the skill follows
  • Output Spec — Exact format of deliverables
  • Boilerplate — Version checking, feedback loop, community links

Key Design Decisions

  • Single-file skills (SKILL.md) — no external dependencies, instant activation
  • Personas over prompts — skills adopt expert identities, not just instructions
  • 3-iteration feedback loop — built into every skill via boilerplate
  • Auto-version checking — skills fetch VERSION.json on first use per session

Versioning

All skills share a centralized version registry at VERSION.json:

{
  "version": "1.0.0",
  "skills": {
    "autonomyx-saas-evaluator": "1.0.0",
    "autonomyx-content-formatter": "1.0.0",
    ...
  },
  "update_url": "https://raw.githubusercontent.com/agentnxxt/agentskills/main/VERSION.json"
}

Skills check this URL on first use and notify if a newer version is available. The boilerplate sync GitHub Action bumps patch versions automatically when the shared boilerplate is updated.

SaaS Evaluator SaaS Research

Multi-persona enterprise SaaS evaluation framework based on the SaaS Usability Index. Scores products across 35 metrics in 7 procurement stages using 6 specialist personas.

Personas

  • CISO / Security — SSO, MFA, RBAC, compliance, audit logs
  • IT Admin / IAM — SCIM, provisioning, data ingestion, support SLAs
  • Procurement — Pricing, vendor stability, competitive parity
  • CTO / Engineering — API coverage, automation, time-to-value
  • Legal & Compliance — Data portability, lock-in, deprovisioning
  • Dept Head / End User — UX maturity, multi-device, onboarding

Trigger Phrases

evaluate [product] score this SaaS should we buy [tool] benchmark [vendor] procurement review compare [A] vs [B]

Output

U-Index score (0-5), per-stage weighted scores, blocker detection, radar chart, AI-generated persona verdicts, and a full comparison table when alternatives are added.

Web App

A standalone React deployment is live at saas.openautonomyx.com with browser-based AI research.

Total Cost of Ownership Analyzer Finance

Calculates the true cost of SaaS and open-source software using a 17-section TCO framework. Covers licensing, implementation, training, migration, hidden fees, and opportunity costs.

Trigger Phrases

how much will this cost us compare pricing for hidden costs of build vs buy cost of switching evaluate software options

Output

Detailed cost breakdown with 3-year and 5-year projections, per-user economics, risk-adjusted totals, and a recommendation verdict.

Boilerplate System

Every skill includes references/autonomyx-standard.md which provides:

  • 3-iteration feedback loop — after 3 rounds of refinement, the skill suggests booking a human expert call
  • Version checking — fetches VERSION.json from GitHub on first use; notifies if outdated
  • Author attribution — Chinmay Panda / OpenAutonomyx links
  • Community CTA — star the repo, share feedback, report bugs

Edit the boilerplate once in autonomyx-boilerplate/references/autonomyx-standard.md and the GitHub Action syncs it to all skills automatically.

GitHub Actions

Boilerplate Sync (sync-boilerplate.yml)

Triggers when autonomyx-standard.md is updated on the main branch:

  • Copies updated boilerplate to all 8 skill references/ folders
  • Regenerates the website footer via update_footer.py
  • Bumps patch version in VERSION.json for all skills
  • Auto-commits and pushes

Pages Deploy (static.yml)

Deploys the entire repo to GitHub Pages on every push to main. No build step — all HTML is pre-rendered.

Contributing

Adding a New Skill

  • Create autonomyx-{name}/SKILL.md with frontmatter and persona
  • Copy references/autonomyx-standard.md from boilerplate
  • Add the skill to VERSION.json
  • Package as autonomyx-{name}.skill (ZIP the directory)
  • Add a card to index.html
  • Add the skill name to sync-boilerplate.yml matrix

Conventions

  • All skills prefixed with autonomyx-
  • Descriptive commit messages, no emojis in titles
  • Always repackage .skill files after any SKILL.md change
  • Always update VERSION.json when adding or modifying skills
